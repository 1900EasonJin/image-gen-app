# 百炼 Qwen/Wan 生图问题分析与修复计划

> 问题发现时间：2026-06-12  
> 涉及文件：`providers/aliyun.js`, `server/routes/generate.js`, `public/js/components/result-grid.js`, `public/js/components/iteration-chain.js`

---

## 1 问题概述

| # | 现象 | 严重程度 |
|---|------|---------|
| 1 | Qwen 系列模型生图结果在前端**完全不渲染**（图片裂） | 🔴 致命 |
| 2 | Qwen 系列迭代链缩略图**全部破裂** | 🔴 致命 |
| 3 | Wan 系列模型生图直接报 **403 错误** | 🔴 致命 |

---

## 2 根因分析

### 2.1 Qwen 图片无法渲染（根因：响应解析器写死了旧格式）

**现状**：`_parseQwenResult()` 方法里，对 `output.choices` 的处理方式是：

```js
// providers/aliyun.js 第 139-142 行
else if (data.output?.choices && Array.isArray(data.output.choices)) {
  results = data.output.choices;
}
```

然后将每个 choice 当作**扁平对象**来提取 URL：

```js
// providers/aliyun.js 第 152 行
const url = r.url || r.image || r.result_url || r;
```

**实际 API 返回格式**（来自百炼官方文档）：

```json
{
  "output": {
    "choices": [
      {
        "finish_reason": "stop",
        "message": {
          "role": "assistant",
          "content": [
            { "image": "https://dashscope-result-xxx.oss-cn-xxx.aliyuncs.com/xxx.png?...", "type": "image" }
          ]
        }
      }
    ]
  }
}
```

图片 URL 的**真实路径**是：

```
choice.message.content[0].image   ← 嵌套在 content 数组里
```

而当前代码找的是：

```
choice.url / choice.image / choice.result_url   ← 全都不存在！
```

**后果链**：
1. `r.image` → `undefined`；`r.url` → `undefined`；兜底 `r` → 整个 choice 对象（非字符串）
2. `typeof url === 'string'` 判断为 `false` → `url` 被置为 `null`
3. 同时 `b64` 也为 `undefined` → `dataUrl` 也为 `null`
4. 返回的 `images` 数组中每个条目 `{ id, url: null, dataUrl: null }`
5. 前端 `resolveImageSrc()` 返回 `''` → `<img>` 的 `src` 为空 → 图片裂

### 2.2 迭代链缩略图破裂（根因：2.1 的连锁反应）

**现状**：`server/routes/generate.js` 保存图片时：

```js
// generate.js 第 62-72 行
for (const img of result.images) {
  const src = img.dataUrl || img.url;
  if (src) {
    // 保存到本地...
    savedImages.push({ ...img, localPath: filePath });
  } else {
    savedImages.push(img);  // ← 没有 localPath！
  }
}
```

因为 `url` 和 `dataUrl` 都是 `null`，走 `else` 分支，`localPath` 为 `undefined`。

存入 session 时：

```js
images: savedImages.map((img) => ({
  id: img.id,
  localPath: img.localPath,  // ← undefined
})),
```

前端迭代链组件渲染时：

```js
// iteration-chain.js
thumb.src = imgId ? `/api/images/${imgId}` : '';  // 走 /api/images/ 路由
// 但图片根本没存到本地 → 404
```

### 2.3 Wan 403 错误（根因：用了异步端点但缺标头）

**现状**：`_generateWan()` 使用的端点是：

```
POST https://dashscope.aliyuncs.com/api/v1/services/aigc/image-generation/generation
```

**对照百炼官方文档**：

| 端点 | 用途 | 必要标头 |
|------|------|---------|
| `POST .../aigc/multimodal-generation/generation` | **同步调用**（Wan2.7/Qwen 均用此） | 无 |
| `POST .../aigc/image-generation/generation` | **异步调用** | **必须带 `X-DashScope-Async: enable`** |

当前代码走了 `image-generation/generation` 但**没有**设置 `X-DashScope-Async: enable` → 服务端拒绝请求 → **403**。

**正确做法**：Wan2.7 同步调用与 Qwen 一样，用 `multimodal-generation/generation` 端点。

## 3 修复计划

### 修改 1：重写 `_parseQwenResult` → 通用图像结果解析

**文件**：`providers/aliyun.js`  
**位置**：`_parseQwenResult` 方法（约第 129 行）

**要做的事**：

```
_parseQwenResult(data) 改名为 _parseImageResult(data)

新逻辑：
1. 优先处理 output.choices[].message.content[] — 这是 Qwen/Wan2.7 的标准同步返回
   遍历 choices → choice.message.content → 收集 { image, type: "image" } 的条目
   
2. 保留旧的 output.results / output.result_urls 兼容逻辑（兜底）

3. 提取时正确拿到 image URL：
   - content[].image → 图片 URL
   - 同时从 usage 中拿 image_count 做校验
```

### 修改 2：Wan 端点切换到同步调用

**文件**：`providers/aliyun.js`  
**位置**：`_generateWan` 方法（约第 145 行）

**要做的事**：

```
1. 将端点从：
   /api/v1/services/aigc/image-generation/generation
   改为：
   /api/v1/services/aigc/multimodal-generation/generation

2. Wan 的 size 参数确认：文档说 Wan2.7 用 "2K"/"1K"/"4K" 格式，
   当前 _normalizeSize → wanSize 的转换逻辑看起来已经正确（"2048*2048" → "2K"）

3. Wan2.7 不支持 prompt_extend 参数，但支持 thinking_mode（默认 true），
   可在 parameters 中显式加 thinking_mode: true

4. 同步调用的请求体格式与 Qwen 完全一致（model/input.messages/parameters）
```

### 修改 3：增强 generate.js 的图片保存容错

**文件**：`server/routes/generate.js`  
**位置**：图片保存循环（约第 62 行）

**要做的事**：

```
1. 当 result.images 中存在 url 时（非 base64），先 fetch 下载再存本地
   当前代码已有此逻辑，确保 url 能被正确提取后走这个流程

2. 加日志：打印每个图片的 url/dataUrl 状态，方便调试

3. 对返回的 result 加校验：如果 images 全部没有 url 也没有 dataUrl，
   向前端返回明确的错误信息（而非静默失败）
```

## 5 附录：百炼 API 关键对比

### 5.1 端点对比

| 模型系列 | 同步端点 | 异步端点 |
|---------|---------|---------|
| Qwen-Image (全系) | `POST .../aigc/multimodal-generation/generation` | N/A（仅 qwen-image-plus/max 支持异步） |
| Wan 2.7 (pro/image) | `POST .../aigc/multimodal-generation/generation` | `POST .../aigc/image-generation/generation` + `X-DashScope-Async: enable` |
| Wan 2.6 | `POST .../aigc/multimodal-generation/generation` | `POST .../aigc/image-generation/generation` + `X-DashScope-Async: enable` |

### 5.2 响应格式对比（同步调用）

Qwen 和 Wan2.7 同步调用的响应格式**完全一致**：

```json
{
  "output": {
    "choices": [
      {
        "finish_reason": "stop",
        "message": {
          "role": "assistant",
          "content": [
            { "image": "https://...", "type": "image" }
          ]
        }
      }
    ]
  },
  "usage": { "image_count": 1, ... }
}
```

### 5.3 size 参数格式

| 模型 | 格式 | 示例 |
|------|------|------|
| Qwen-image-2.0 系列 | `"宽*高"` | `"2048*2048"`, `"2688*1536"` |
| Qwen-image-max/plus | 5 种固定预设 | `"1664*928"`, `"1328*1328"` 等 |
| Wan 2.7 系列 | 缩写 或 `"宽*高"` | `"2K"`, `"4K"`, `"1024*1024"` |
