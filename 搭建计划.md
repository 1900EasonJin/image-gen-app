## 1. 项目概述

### 1.1 目标

打造一个 **macOS 原生体验** 的 AI 图片生成桌面应用，具备以下核心能力：

| 能力 | 说明 |
|------|------|
| 🔌 动态接入 | 输入 API Key 后自动拉取服务商可用的生图模型 |
| 🎨 文生图 | 通过提示词生成图片 |
| 🔄 图生图 | 不满意可基于已生成的图片继续修改 |
| 📜 迭代历史 | 保留每次生成和修改的完整链条 |
| 💻 CLI 支持 | 可在终端 / Pi Agent / OpenCode 中通过 skill 调用 |
| 🖥️ Notion 风格 | 紧凑、干净、优雅的前端界面 |

### 1.2 与旧版的区别

| 维度 | 旧版 | 新版 |
|------|------|------|
| 模型管理 | 硬编码在 `config/models.js` | **动态拉取**，输入 API Key 即获取最新模型 |
| UI 风格 | 传统侧边栏+大区块 | **Notion 风格**，紧凑轻量 |
| 图片交互 | 悬停显示下载/改图按钮 | 悬停显示 **三个选项**：预览全图、下载、继续修改 |
| 预览 | 无内置预览 | 点击即可**前端全屏渲染原图** |
| 迭代 | 简单改图 | 保留**完整迭代链条**（原图 → 修改1 → 修改2...） |
| Provider | 仅火山方舟+阿里云百炼 | **可扩展架构**，新增 Provider 只需添加一个文件 |
| 代码结构 | 单文件混杂 | 模块化、分层清晰 |

## 2. 技术选型

### 2.1 推荐方案

| 层级 | 技术 | 理由 |
|------|------|------|
| 桌面壳 | **Electron** | 旧项目已验证可行；macOS 集成成熟（Dock 图标、窗口管理、原生菜单） |
| 前端 | **Vanilla HTML + Modern CSS + ES Modules** | 无构建步骤、零依赖前端、易于理解维护；Notion 风格完全用 CSS 实现 |
| CSS 方案 | **CSS Custom Properties + 系统字体栈** | 统一主题变量，Inter / SF Pro 系统字体，无需引入字体文件 |
| 后端 | **Express 内嵌于 Electron 主进程** | 旧项目验证过的自包含架构；避免子进程残留、端口冲突 |
| HTTP 客户端 | **Node 内置 `fetch`（Node 18+）** | 零依赖，原生支持 |
| CLI | **Node.js 脚本 + Commander.js** | 轻量 CLI 框架，比手写参数解析更可靠 |
| API Key 加密 | **Node `crypto`（AES-256-GCM）+ PBKDF2** | 内置模块，无需额外依赖；GUI 和 CLI 共享同一加密配置文件 |
| 运行方式 | **开发模式 `npm start`** | 先用起来，暂不打 `.app` 包，后续需要再配置 electron-builder |

### 2.2 不推荐但可考虑的替代方案

| 方案 | 优点 | 缺点 | 为什么不选 |
|------|------|------|------------|
| React + Vite | 组件化、生态丰富 | 需要构建步骤、依赖重 | 用户非专业前端，维护成本高 |
| Vue 3 | 学习曲线低 | 仍需构建 | 同上 |
| Tauri | 体积小 | Rust 依赖、生态不成熟 | 调试复杂，不适合快速迭代 |
| PWA | 跨平台 | 没有 macOS 原生体验、无法设 Dock 图标 | 用户体验降级 |

### 2.3 关键决策

- **不使用前端框架**：项目规模中等，vanilla JS 足够；Notion 风格主要靠 CSS 实现
- **不引入构建工具**：零 `webpack/vite`，直接用 `ES Modules` + `importmap` 或简单的 script 标签加载
- **CSS 使用原生嵌套/Custom Properties**：现代浏览器完全支持，无需 Sass/Less
- **Node 版本要求 ≥ 18**：使用原生 `fetch`，避免引入 `node-fetch` 或 `axios`
- **API Key 加密存储**：使用 AES-256-GCM 加密，密钥由 PBKDF2 从机器标识派生。GUI（Electron）和 CLI 共享同一套加密逻辑和配置文件（`~/.image-gen-v2/config.json`），两者均可读写

### 3.2 数据流

```
用户输入 API Key
  → 前端 POST /api/provider/connect
    → 后端 Provider.listModels(apiKey)
      → 返回可用模型列表
        → 前端渲染模型选择器

用户选择模型 + 输入 Prompt
  → 前端 POST /api/generate
    → 后端 Provider.generate(model, prompt, params)
      → 返回图片 Base64/URL
        → 前端展示 + 存入迭代历史

用户点击"继续修改"
  → 前端 POST /api/generate (附带 referenceImage)
    → 后端 Provider.img2img(model, referenceImage, prompt)
      → 返回新图片
        → 追加到迭代链条
```

### 3.3 Provider 插件化设计

每个 Provider 是一个 ES Module，导出统一接口：

```js
// providers/volcengine.js
export default {
  id: 'volcengine',
  name: '火山方舟 (Seedream)',
  
  // 连接测试 + 拉取模型列表
  async connect(apiKey) {
    // 调用 API 获取模型列表
    // 返回 { success: true, models: [...] }
  },
  
  // 文生图
  async text2image({ model, prompt, n, size, apiKey }) {
    // 返回 { images: [{ base64, url }], usage: {...} }
  },
  
  // 图生图
  async img2img({ model, prompt, referenceImage, n, size, apiKey }) {
    // 返回同上
  },
  
  // 默认参数
  defaults: { n: 1, size: '2K' }
}
```

新增 Provider 只需：在 `providers/` 下新建文件 → 实现接口 → 在 `providers/index.js` 注册。

## 5. 后端 API 设计

### 5.1 Express 路由

所有 API 前缀 `/api`，端口 `3456`：

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/providers` | 获取所有可用 Provider 列表 |
| `POST` | `/api/providers/:id/connect` | 连接 Provider（传入 API Key），返回可用模型 |
| `POST` | `/api/generate` | 文生图 / 图生图 |
| `GET` | `/api/generate/:id/status` | 查询异步生成任务状态（预留） |
| `GET` | `/api/images/:id` | 获取已生成图片（从本地缓存） |
| `DELETE` | `/api/images/:id` | 删除图片 |
| `GET` | `/api/sessions` | 获取历史会话列表 |
| `GET` | `/api/sessions/:id` | 获取单个会话详情（含迭代链条） |
| `DELETE` | `/api/sessions/:id` | 删除会话 |

### 5.2 请求/响应格式

#### POST `/api/providers/:id/connect`

```json
// Request
{ "apiKey": "sk-xxxx" }

// Response
{
  "success": true,
  "provider": { "id": "volcengine", "name": "火山方舟" },
  "models": [
    { "id": "doubao-seedream-5-0-260128", "name": "Seedream 5.0 Lite", "desc": "最新旗舰", "maxN": 4, "sizes": ["2K", "1K"] },
    ...
  ]
}
```

#### POST `/api/generate`

```json
// Request（文生图）
{
  "provider": "volcengine",
  "model": "doubao-seedream-5-0-260128",
  "prompt": "一只穿宇航服的柴犬",
  "n": 2,
  "size": "2K",
  "sessionId": "ses_xxx"  // 可选，追加到已有会话
}

// Request（图生图）
{
  "provider": "volcengine",
  "model": "doubao-seedream-5-0-260128",
  "prompt": "把背景改成日落",
  "referenceImage": "data:image/png;base64,xxx"  // 或本地路径
}

// Response
{
  "success": true,
  "images": [
    { "id": "img_001", "dataUrl": "data:image/png;base64,...", "index": 0 },
    ...
  ],
  "sessionId": "ses_xxx",
  "iterationIndex": 2  // 当前迭代序号
}
```

## 6. Provider 接入机制

### 6.1 首批支持的 Provider

| Provider | 服务商 | 模型系列 | API 地址 |
|----------|--------|----------|----------|
| `volcengine` | 火山方舟 | Seedream 全系列 | `https://ark.cn-beijing.volces.com/api/v3` |
| `aliyun` | 阿里云百炼 | Qwen-Image（含 Wan 系列） | `https://dashscope.aliyuncs.com/api/v1` |

### 6.2 动态模型拉取

**火山方舟**：
```
GET https://ark.cn-beijing.volces.com/api/v3/models
Authorization: Bearer {apiKey}
→ 过滤 type=image 的模型
```

**阿里云百炼（DashScope）**：
```
Qwen-Image 和 Wan 系列模型均通过 DashScope API 调用。
模型列表可通过百炼控制台 API 获取，或从文档同步维护。
```

> ⚠️ 注意：如果服务商没有标准化的模型列表 API，则退化为本地配置 + 定期从文档同步。Wan 模型通过阿里云百炼统一入口接入，不需要单独 Provider。

### 6.3 如何添加新 Provider

1. 在 `providers/` 下新建文件，如 `providers/openai.js`
2. 导出实现统一接口的对象
3. 在 `providers/index.js` 中注册

示例骨架：

```js
// providers/openai.js
export default {
  id: 'openai',
  name: 'OpenAI (DALL·E)',
  
  async connect(apiKey) {
    // OpenAI 没有模型列表 API，返回固定模型
    return {
      success: true,
      models: [
        { id: 'dall-e-3', name: 'DALL·E 3', desc: '最新版', maxN: 1, sizes: ['1024x1024', '1792x1024'] }
      ]
    };
  },
  
  async text2image({ model, prompt, n, size, apiKey }) {
    const resp = await fetch('https://api.openai.com/v1/images/generations', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ model, prompt, n, size })
    });
    const data = await resp.json();
    return {
      images: data.data.map((img, i) => ({
        id: `img_${Date.now()}_${i}`,
        url: img.url,
        index: i
      }))
    };
  },
  
  // 如不支持图生图，可以不实现
  async img2image({ model, prompt, referenceImage, n, size, apiKey }) {
    throw new Error('DALL·E 暂不支持图生图');
  }
};
```

## 7. CLI 命令行工具

### 7.1 设计目标

兼容旧版 CLI 的用法，同时新增便捷命令。

### 7.2 命令设计

```bash
# 查看所有 Provider
image-gen providers

# 连接 Provider 并列出模型
image-gen connect volcengine --key "your-key"
image-gen models --provider volcengine

# 文生图
image-gen generate \
  --provider volcengine \
  --model doubao-seedream-5-0-260128 \
  --prompt "一只穿宇航服的柴犬" \
  --count 2 \
  --size 2K \
  --output ~/Downloads/

# 图生图
image-gen generate \
  --provider volcengine \
  --model doubao-seedream-5-0-260128 \
  --prompt "把背景改成日落" \
  --image ~/Pictures/original.png \
  --output ~/Downloads/

# JSON 输出（供 Pi Agent 解析）
image-gen generate ... --json

# 保存 API Key
image-gen config set-key volcengine "your-key"
image-gen config set-key aliyun "sk-your-key"

# 查看配置
image-gen config show
```

### 7.3 实现

使用 `commander` 库（约 30KB），提供友好的 help 输出和错误提示。

```js
#!/usr/bin/env node
import { program } from 'commander';
// ...
```

安装方式：
```bash
npm link  # 在项目目录下执行，全局注册 image-gen 命令
```

## 8. Pi Agent Skill 集成

### 8.1 Skill 设计

在项目 `skills/image-gen/` 下创建 SKILL.md 和辅助脚本，使 Pi Agent 可以通过对话调用生图能力。

### 8.2 Skill 工作流

1. 用户说"帮我生成一张xxx"
2. Pi Agent 解析意图 → 提取 prompt、模型偏好
3. 调用 CLI（`--json`）执行生成
4. 解析 JSON 返回值
5. 将图片路径反馈给用户

### 8.3 Skill 文件

```
skills/image-gen/
├── SKILL.md              ← Pi Agent skill 定义
├── generate.sh           ← Shell 封装脚本
└── references/
    └── models.md         ← 模型速查表（自动从 Provider 导出）
```

## 12. API Key 安全设计

### 12.1 加密方案

使用 **AES-256-GCM** 对称加密，密钥通过 **PBKDF2** 从机器标识派生：

```
┌────────────────────────────────────────────────┐
│                  加密流程                        │
│                                                 │
│  机器标识 (hostname + MAC + username)            │
│       │                                         │
│       ▼                                         │
│  PBKDF2 (salt + 100000 迭代)                    │
│       │                                         │
│       ▼                                         │
│  256-bit 密钥 ──→ AES-256-GCM 加密              │
│                       │                         │
│                       ▼                         │
│  { iv, tag, ciphertext } → Base64 → config.json │
└────────────────────────────────────────────────┘
```

- **存储位置**：`~/.image-gen-v2/config.json`
- **加密内容**：所有 API Key（每个 Provider 的 key 单独加密）
- **GUI 和 CLI 共享**：同一套 `lib/crypto-store.js` 模块
- **不依赖 Electron safeStorage**：纯 Node `crypto` 实现，CLI 也能用

### 12.2 配置文件结构

```json
{
  "providers": {
    "volcengine": {
      "encrypted": true,
      "key": "aGVsbG8gd29ybGQ...base64",
      "connectedAt": "2026-06-11T10:00:00Z"
    },
    "aliyun": {
      "encrypted": true,
      "key": "d29ybGQgaGVsbG8...base64"
    }
  },
  "preferences": {
    "defaultProvider": "volcengine",
    "theme": "system"
  }
}
```

### 12.3 安全边界

- 加密密钥源自本机信息，换机器后配置文件不可解密（需重新输入 API Key）
- 配置文件权限设为 `600`（仅所有者可读写）
- 解密后的 Key 仅在内存中存在，不落盘
