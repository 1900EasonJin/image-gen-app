# Image Gen 工作区改造计划

## 一、现状分析

### 1.1 当前界面
左侧 240px Sidebar（Image Gen 标题 + 模型选择 + 历史会话），右侧 `main-content` 完全空白，没有充分利用工作区空间。

### 1.2 参考图设计
右侧工作区应包含：
- **顶部工具栏**：左侧状态标签（如「暂无数据」），中间「绘图 / 编辑」Tab 切换，右侧操作按钮
- **中央画布区**：大面积图片展示区域，空状态显示「暂无图片」
- **右侧垂直工具栏**：「+」按钮（虚线边框，上传/添加参考图）、另一个功能按钮（绿色边框）
- **底部输入区**：文本输入框，placeholder 提示「输入你的图片描述，文本绘制用"双引号"包裹」，右下角「文 A」图标和绿色发送按钮

### 1.3 源码结构
```
public/
  index.html              — 主页面结构
  css/
    variables.css         — CSS 变量
    layout.css            — 整体布局
    components.css        — 组件样式
    lightbox.css          — 全屏预览
  js/
    app.js                — 主入口，事件调度
    state.js              — 全局状态
    api.js                — API 接口
    components/
      sidebar.js          — 侧边栏（模型选择）
      input-area.js       — 底部输入区 + 生成逻辑
      result-grid.js      — 画布图片网格渲染
      iteration-chain.js  — 迭代历史
      lightbox.js         — 全屏预览
      settings.js         — 设置弹窗
    utils/
      dom.js              — DOM 工具
      toast.js            — Toast 通知
```

---

## 二、改造目标

将右侧 `main-content` 改造成一个功能完整的工作区（Work Area），包含：
1. 顶部模式切换（绘图 / 编辑）
2. 中央画布（图片展示 + 空状态）
3. 右侧垂直工具栏
4. 底部输入区（风格统一）

### 步骤 3：JS 组件逻辑改造

#### 3.1 新建组件 `js/components/work-area.js`

```javascript
import state from '../state.js';
import { $ } from '../utils/dom.js';

const tabDraw = $('#tabDraw');
const tabEdit = $('#tabEdit');
const statusTag = $('#statusTag');
const btnAddRef = $('#btnAddRef');

export function init() {
  // 模式 Tab 切换
  tabDraw?.addEventListener('click', () => switchMode('draw'));
  tabEdit?.addEventListener('click', () => switchMode('edit'));

  // 添加参考图按钮（右侧工具栏）
  btnAddRef?.addEventListener('click', handleAddReference);

  // 监听生成状态变化
  window.addEventListener('imagesGenerated', () => {
    statusTag.textContent = '已生成';
    statusTag.style.color = 'var(--success)';
  });

  window.addEventListener('stateChange', (e) => {
    if (e.detail.generating !== undefined) {
      statusTag.textContent = e.detail.generating ? '生成中...' : (state.currentSession ? '已生成' : '暂无数据');
    }
  });
}

function switchMode(mode) {
  state.workMode = mode;

  // 更新 Tab 样式
  [tabDraw, tabEdit].forEach((tab) => {
    tab.classList.toggle('active', tab.dataset.mode === mode);
  });

  // 触发模式切换事件
  window.dispatchEvent(new CustomEvent('modeSwitch', { detail: { mode } }));

  // 更新输入框 placeholder
  const promptInput = $('#promptInput');
  if (mode === 'edit') {
    promptInput.placeholder = '描述你想要修改的部分...';
  } else {
    promptInput.placeholder = '输入你的图片描述，文本绘制用"双引号"包裹';
  }
}

function handleAddReference() {
  // 创建隐藏的文件 input
  const input = document.createElement('input');
  input.type = 'file';
  input.accept = 'image/*';
  input.onchange = async (e) => {
    const file = e.target.files[0];
    if (!file) return;

    const reader = new FileReader();
    reader.onload = () => {
      const dataUrl = reader.result;
      state.referenceImage = dataUrl;

      // 显示参考图缩略图
      const thumb = $('#refImageThumb');
      const preview = $('#refImagePreview');
      if (thumb && preview) {
        thumb.src = dataUrl;
        preview.classList.remove('hidden');
      }

      // 切换到编辑模式
      switchMode('edit');
    };
    reader.readAsDataURL(file);
  };
  input.click();
}

export function getActiveMode() {
  return state.workMode || 'draw';
}
```

export async function generateImage({ provider, model, prompt, n, size, referenceImage, sessionId, mode }) {
  const resp = await fetch(`${BASE_URL}/api/generate`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      provider,
      model,
      prompt,
      n,
      size,
      referenceImage: referenceImage || undefined,
      sessionId: sessionId || undefined,
      mode: mode || 'draw', // 新增
    }),
  });
  return resp.json();
}

## 五、交互逻辑梳理

### 5.1 用户操作流程

1. **打开应用**：右侧工作区显示「暂无数据」标签 + 空画布「暂无图片」
2. **选择模型**：左侧 sidebar 选择模型后，model-badge 更新
3. **输入 prompt**：底部输入框输入，点击绿色发送按钮或 ⌘+Enter
4. **生成中**：状态标签变为「生成中...」，画布显示 loading 遮罩
5. **生成完成**：状态标签变为「已生成」，画布显示图片网格
6. **点击图片**：弹出 Lightbox 预览
7. **点击修改**：Lightbox 关闭 → 进入编辑模式（Tab 切换到「编辑」+ 输入框 placeholder 变化 + 参考图缩略图显示）
8. **添加参考图**：点击右侧工具栏「+」按钮，上传图片，自动切换到编辑模式
9. **新建对话**：左侧「+」按钮，清空画布回到空状态

### 5.2 模式切换逻辑

| 模式 | Tab 样式 | 输入框 placeholder | 参考图 |
|------|----------|-------------------|--------|
| 绘图 | 绘图 Tab 高亮 | 输入你的图片描述... | 隐藏 |
| 编辑 | 编辑 Tab 高亮 | 描述你想要修改的部分... | 显示（如有） |

### 5.3 事件流向

```
用户点击 Tab → work-area.js switchMode() → 更新 state.workMode
                                        → 更新 DOM Tab 样式
                                        → 更新 placeholder
                                        → 触发 CustomEvent('modeSwitch')

用户点击生成 → input-area.js handleGenerate() → 读取 state.workMode
                                           → 调用 api.generateImage({mode})
                                           → 触发 imagesGenerated
                                           → work-area.js 监听并更新状态标签

用户上传参考图 → work-area.js handleAddReference() → 读取文件为 dataURL
                                                → 设置 state.referenceImage
                                                → 显示缩略图
                                                → 自动切换到 edit 模式
```
